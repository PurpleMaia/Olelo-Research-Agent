# Implementation Plan: Agentic Papakilo Search Integration

**Branch**: `002-agentic-search` | **Date**: 2026-02-20 | **Spec**: `specs/002-agentic-search/spec.md`
**Input**: Integrate `Research_agentic_tool/` as a live search source in the research pipeline

---

## Summary

Integrate the Playwright-based Papakilo Database scraper (currently a standalone tool in `Research_agentic_tool/`) as a live search source within the app's existing research pipeline. When a user submits a research query, the system will now run Papakilo newspaper searches in parallel with the existing vector search, merge results, and synthesize findings using Claude. This gives researchers access to live Hawaiian newspaper archives without leaving the app.

---

## Technical Context

**Language/Version**: TypeScript 5 / Node.js 20 (Next.js 16)
**Primary Dependencies**: `playwright-core`, existing `@anthropic-ai/sdk`, `kysely`, `pg`
**Storage**: No new tables — existing `research_sessions.results` JSONB holds merged results
**Testing**: Manual integration test script (`scripts/test-papakilo-search.ts`)
**Target Platform**: Node.js server (Next.js API routes — not Edge runtime)
**Project Type**: Web application (Next.js App Router)
**Performance Goals**: Papakilo search completes within 30s; total pipeline latency budget unchanged from user perspective (Papakilo runs in parallel with vector search)
**Constraints**: Respectful scraping (sequential requests, 1-2s delay between terms); Chrome must be installed; Papakilo failures are non-blocking
**Scale/Scope**: Single-user sessions; top 3 search terms × top 3 articles = up to 9 article fetches per research session

## Implementation Phases

### Phase 1: Papakilo Service (`src/services/research/papakilo.ts`)

Port the logic from `Research_agentic_tool/.../search-newspapers.js` and `view-article.js` into a typed TypeScript module:

1. `searchPapakilo(term)` — launches headless Chrome, searches papakilodatabase.com, returns article list
2. `fetchArticleContent(url)` — navigates to article URL, extracts OCR text
3. `researchWithPapakilo(searchTerms, options)` — orchestrates multi-term search + selective article fetch; never throws

**Key implementation details from existing scripts:**
- URL: `https://www.papakilodatabase.com/pdnupepa/cgi-bin/pdnupepa?a=p&p=home`
- Search input selector: `#homepagesearchinputtxq`
- Article links: `a[href*="a=d&d="]`
- Wait strategy: `waitUntil: 'domcontentloaded'` + 2-5s explicit waits (JavaScript-heavy site)
- Browser config: `{ headless: true, channel: 'chrome' }`

### Phase 2: Type Updates (`src/types/research.ts`)

Add `'papakilo-live'` to `Source.type` union. No other type changes.

### Phase 3: Config (`src/lib/config/research.ts`)

Add `papakiloEnabled`, `papakiloMaxTerms`, `papakiloTimeoutMs` with defaults.

### Phase 4: Orchestrator (`src/services/research/orchestrator.ts`)

Modify `execute()` to run Papakilo search in parallel with vector search using `Promise.allSettled()`. Merge results into unified context before synthesis. Add stream activity messages for Papakilo steps.

### Phase 5: Synthesis prompt update (`src/services/research/claude.ts`)

Update `SYNTHESIS_PROMPT` to handle `papakilo-live` sources — remind Claude these are live-scraped with OCR noise, include article URLs in source attribution.

### Phase 6: Install dependency

```bash
npm install playwright-core
npx playwright install chromium  # if Chrome not already available
```

### Phase 7: Test

Create `scripts/test-papakilo-search.ts` to verify Playwright search works in isolation before integration testing.

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| papakilodatabase.com down/rate-limited | `Promise.allSettled` + non-blocking failure path; user sees graceful degradation message |
| Chrome not installed on server | Document in quickstart.md; `PAPAKILO_SEARCH_ENABLED=false` as escape hatch |
| Playwright adds significant bundle size | `playwright-core` is server-only; not included in client bundle |
| Slow Papakilo search increases perceived latency | Runs in parallel with vector search; first activity messages appear immediately |
| OCR noise in Papakilo results | `SYNTHESIS_PROMPT` updated to handle OCR artifacts; confidence levels applied |
| Site structure changes break selectors | Selectors isolated in `papakilo.ts`; easy to update in one place |
